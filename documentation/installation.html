Installation

Installing GENFIRE from Binaries

The easiest way to use GENFIRE is to download the appropriate precompiled binary for your operating system. These consist of just a single executable file that contains all of the code. There are 64-bit versions for Mac OS X 10.11, Windows 10, and Ubuntu 10.04. It's possible that the binaries may work on other versions of these operating systems, but as these are the three that I use at the time of writing this document so they are the three that have been tested.

Installing GENFIRE as a Python package

To install GENFIRE as a python package, first get the source code from the git repository. Either download the code as a compressed file or clone the repository with

$ git clone "some address that I need to change"

From here the instructions depend on your OS

Mac OS X

Python is preinstalled on Mac OS X, but it is generally a bad idea to alter the system python in /usr/bin as some programs depend on it. I would highly recommend using the package manager homebrew to create a python environment for you. The following commands will install homebrew and python

$ /usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"
$ brew install python

Once that's done, navigate to the top source code directory. You should be in the same folder as requirements.txt and setup.py

$ cd /path/to/GENFIRE

Install (most of the) dependencies using pip

$ pip install --upgrade pip
$ pip install -r requirements.txt

You will also need sip and PyQt4, which unfortunately cannot be installed with pip. They're easy to install using homebrew.

$ brew install sip
$ brew install pyqt

You should now have all the dependencies necessary to run GENFIRE. To install it, make sure you are in the directory with setup.py and enter

$ python setup.py install

If everything worked, you can now launch the gui from the command line with

$ genfire

You can also use GENFIRE from python

$ python
$ import GENFIRE


Linux (Ubuntu 14.04)

Navigate to the source code directory

$ cd /path/to/GENFIRE

Install (most of the) dependencies using pip

$ pip install --upgrade pip
$ pip install -r requirements.txt

You will also need sip and PyQt4, this command should install both.

$ sudo apt-get install python-qt4

You should now have all the dependencies necessary to run GENFIRE. To install it, make sure you are in the directory with setup.py and enter

$ python setup.py install

If everything worked, you can now launch the gui from the command line with

$ genfire

You can also use GENFIRE from python

$ python
$ import GENFIRE


Windows 10

For python on Windows 10 I would recommend using Anaconda from Continuum Analytics. It's a distribution of python that contains most of the packages used by GENFIRE all wrapped into a simple-to-use installer. Once you have python setup, open up a cmd prompt and navigate to the source directory. You should be in the same folder as requirements.txt and setup.py

$ chdir C:\path\to\GENFIRE
$ import GENFIRE

Install (most of the) dependencies using pip

$ C:\path\to\Anaconda\Scripts\pip install --upgrade pip
$ C:\path\to\Anaconda\Scripts\pip install -r requirements.txt

You will also need sip and PyQt4. Download the binary installer for your version of Windows.

Once you have installed the dependencies, install GENFIRE with

$ C:\path\to\Anaconda\python setup.py install

If everything worked you can now use GENFIRE from a python console.

$ python
$ import GENFIRE
In principle you should also be able to run GENFIRE from the terminal with

$ genfire

however, this type of bash executable doesn't appear to work the same is it does for Mac/Linux. Although my intentions were pure, at this point I must admit I am not a Windows user. If any of you who are much wiser than I know a solution, feel free to email me.

Instead, you can just run the gui as a python script

$ C:\path\to\Anaconda\python \path\to\GENFIRE\GENFIRE\gui\launch.py



Installation Troubleshooting

If you have trouble installing PyQt4 or sip, consult their documentation
If you have some problem with the "pip install -r requirements.txt" step, you can view the requirements.txt file to see the packages that are necessary, and try to install them one-by-one.
